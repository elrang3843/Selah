User Guide

What is Selah?

Selah is a Windows desktop application for worship music preparation, project-based audio editing, and AI-assisted stem separation. It is designed for small churches, mission fields, and worship teams to prepare accompaniment-oriented audio from recordings or worship media. Selah is free and open-source (GPLv3) and runs entirely offline.

Getting Started

When you first launch Selah, the welcome screen is displayed. Click ＋ New Project to create a new project, or 📂 Open Project to load an existing one. Choose a name and sample rate — 48 kHz is recommended for video/streaming; 44.1 kHz for music files.

Managing Projects

A project saves all tracks, clips, and settings in a single folder.

Ctrl+S Save  |  Ctrl+Shift+S Save As  |  Ctrl+O Open

When closing with unsaved changes, you will be prompted to save.

Importing Audio

Use File → Import Audio/Video to import WAV, MP3, FLAC, MP4, and other formats. Formats other than WAV require FFmpeg. Selecting multiple files at once adds each as a separate track.

Timeline Editing

Move clip — Drag a clip to change its position.
Split clip — Position the playhead and press S.
Delete clip — Select a clip and press Delete.
Zoom — Use Ctrl+Wheel to zoom in and out.
Snap — Press N to toggle snapping.

Use the volume slider, M (Mute), and S (Solo) buttons in each track header to control individual tracks.

AI Stem Separation / MR Extraction

Use AI stem separation to remove vocals or isolate drums, bass, and instruments.

1. Open Tools → Model Manager and install a model. (Requires Python 3.10+ and internet)
2. Select a clip in the timeline.
3. Run Edit → Separate Stems.
4. Each stem is automatically added as a new track.

Noise Reduction: Edit → Reduce Noise. (Requires: pip install noisereduce)

Import Sheet Music

Automatically generate per-instrument audio tracks from a scanned or photographed score image (PNG, JPG, TIFF).

1. Choose File → Import Sheet Music.
2. Select a score image file, then click Recognize.
3. After recognition, select the instruments you want. (★ = auto-recommended)
4. Click Insert into Track — a new track is added for each instrument.

This feature requires Python packages (oemer, music21, etc.), FluidSynth, and a SoundFont (.sf2). See Required Tools ⑦⑧⑨ below.

Exporting

Use File → Export as WAV (Ctrl+E) to render the entire timeline to a 24-bit WAV. Muted tracks are excluded. If the metronome is on, it will be included — turn it off before exporting.

Required Tools Setup

The following tools are required for stem separation and noise reduction. Basic WAV editing works without any additional installation.

① Install Python — Required (all MR extraction)

Download Python 3.10+ from python.org/downloads. Check "Add Python to PATH" during install. Restart Selah after installation.

⚠ Do not use the Microsoft Store version of Python — it does not work correctly as a background process.
Verify (Command Prompt):
python --version

② Install ONNX Runtime — Required (Demucs model)

pip install onnxruntime numpy scipy
If pip is not found:
python -m pip install onnxruntime numpy scipy
⚠ Virtual environments (venv, conda): Selah does not auto-detect virtual environments. Install to global Python, or use py -m pip install.

③ Install audio-separator — Recommended (vocal separation)

Recognizes a wider range of voices than Demucs. For GPU acceleration, replace [cpu] with [gpu]. (Requires CUDA 12.x)

pip install "audio-separator[cpu]"

④ Install noisereduce — Required (noise reduction)

pip install noisereduce

⑤ Install FFmpeg — Required (MP3/MP4 import)

Method A — Open Command Prompt as Administrator (recommended):

winget install Gyan.FFmpeg

Method B (manual) — Download from ffmpeg.org/download.html and add the bin\ folder to your system PATH.

⑥ Download AI Separation Model

After installing the tools above, open Tools → Model Manager to download a model.

Hybrid Transformer Demucs (htdemucs)
4-stem separation · 303 MB · requires ONNX Runtime
UVR MDX-NET Vocal FT
vocal/backing 2-stem · 65 MB · requires audio-separator

Select a model, click Install Model / Download, then click Refresh when done.

⑦ Install OMR packages — Required (sheet music import)

pip install oemer music21 mido Pillow scipy
oemer downloads its own model weights automatically on first run (internet required).

⑧ Install FluidSynth — Required (sheet music → audio)

Both steps are required.

Step 1 — Install the native library (libfluidsynth-3.dll):

choco install fluidsynth

Or download the Windows installer from fluidsynth.org. Selah detects the DLL automatically — no PATH change needed.

Chocolatey may install to C:\Program Files\FluidSynth\bin\ or C:\tools\fluidsynth\bin\ depending on the version. Selah searches both paths automatically.

Step 2 — Install the Python wrapper:

pip install fluidsynth
⚠ Installing only the pip package without the native installer will cause synthesis errors. Both steps are required.

Troubleshooting — "FLUIDSYNTH_DLL_MISSING" error: If you see this error during synthesis, Selah cannot find libfluidsynth-3.dll. Run the following command in an Administrator PowerShell to locate the DLL:

Get-ChildItem -Path C:\ -Filter "libfluidsynth*.dll" -Recurse -ErrorAction SilentlyContinue | Select-Object -ExpandProperty FullName

If a DLL is found, add its folder (e.g. C:\tools\fluidsynth\bin) to the system PATH, then restart Selah.

⑨ Install SoundFont (.sf2) — Required (instrument tones)

A SoundFont is an instrument sample bank FluidSynth uses to render MIDI to audio.

Recommended free SoundFont: GeneralUser GS (~29 MB)
Download from schristiancollins.com/generaluser.php, then copy
GeneralUser GS.sf2 to:

%AppData%\Selah\soundfonts\
Type or paste this path directly into Windows Explorer's address bar to open the folder.

Ethical Use Guidelines

Selah was created to support worship ministry. Users are responsible for verifying the copyright status of any audio they process.

Purpose

Selah is a ministry-oriented audio software project designed to support Christian worship preparation, small church audio workflows, and missionary accompaniment preparation. It prioritizes non-commercial, service-oriented, and local-environment use cases.

Respect for Rights and Ownership

Users must respect the rights of composers, lyricists, performers, churches, ministries, publishers, and copyright holders. The existence of technical functionality does not automatically mean a use is legally or ethically permitted.

Examples of Responsible Use

✓ Using accessible worship audio to prepare backing tracks for a small worship team
✓ Organizing ministry recordings for internal church rehearsal
✓ Testing stem separation on audio you have permission to process

Examples of Prohibited Use

✗ Redistributing copyrighted worship recordings without permission
✗ Commercially repackaging separated stems
✗ Removing vocals or extracting tracks without lawful rights
✗ Misrepresenting official church or ministry approval

AI / Model Caution

Model outputs may be imperfect and may contain artifacts. Third-party model terms may impose additional restrictions. Maintainers do not guarantee suitability for commercial distribution or public release.
