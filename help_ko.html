사용 가이드

셀라란?

Selah는 예배 음악 준비, 프로젝트 기반 오디오 편집, AI 스템 분리를 위한 Windows 데스크탑 애플리케이션입니다. 소형 교회, 선교지, 소규모 찬양팀이 녹음이나 예배 영상에서 반주 중심의 오디오 자료를 실용적으로 준비할 수 있도록 설계되었습니다. 인터넷 연결 없이 로컬에서 실행되는 무료·오픈소스(GPLv3) 도구입니다.

시작하기

Selah를 처음 실행하면 환영 화면이 표시됩니다. ＋ 새 프로젝트 버튼으로 새 프로젝트를 만들거나, 📂 프로젝트 열기로 기존 프로젝트를 불러올 수 있습니다. 새 프로젝트 생성 시 이름과 샘플레이트를 선택합니다. 영상·스트리밍 작업에는 48 kHz, 음악 파일 중심이면 44.1 kHz를 권장합니다.

프로젝트 관리

프로젝트는 모든 트랙, 클립, 설정을 한 폴더에 저장합니다.

Ctrl+S 저장  |  Ctrl+Shift+S 다른 이름으로 저장  |  Ctrl+O 열기

창을 닫을 때 저장하지 않은 변경사항이 있으면 저장 여부를 확인합니다.

오디오 가져오기

파일 → 오디오/영상 불러오기로 WAV, MP3, FLAC, MP4 등 다양한 형식을 가져올 수 있습니다. WAV 이외 형식은 FFmpeg가 필요합니다. 여러 파일을 한 번에 선택하면 각각 별도 트랙으로 추가됩니다.

타임라인 편집

클립 이동 — 클립을 드래그해 위치를 변경합니다.
클립 분할 — 재생 커서를 원하는 위치에 놓고 S 키를 누릅니다.
클립 삭제 — 클립 선택 후 Delete 키를 누릅니다.
줌 — Ctrl+Wheel로 확대·축소합니다.
스냅 — N 키로 켜고 끌 수 있습니다.

트랙 헤더에서 볼륨 슬라이더, M(음소거), S(솔로) 버튼으로 각 트랙을 조절합니다.

AI 스템 분리 / MR 추출

AI 스템 분리로 음악에서 보컬을 제거하거나 드럼·베이스·악기를 분리할 수 있습니다.

1. 도구 → 모델 관리자를 열고 분리 모델을 설치합니다. (Python 3.10+, 인터넷 연결 필요)
2. 분리할 클립을 타임라인에서 선택합니다.
3. 편집 → 스템 분리를 실행합니다.
4. 분리 완료 후 각 스템이 새 트랙으로 자동 추가됩니다.

노이즈 감소: 편집 → 노이즈 감소. (pip install noisereduce 필요)

악보 이미지 가져오기

스캔하거나 촬영한 악보 이미지(PNG·JPG·TIFF)에서 악기별 오디오 트랙을 자동으로 생성합니다.

1. 파일 → 악보 이미지 가져오기를 선택합니다.
2. 악보 이미지 파일을 선택하고 인식 시작을 클릭합니다.
3. 인식 완료 후 악기 목록에서 원하는 악기를 선택합니다. (★ 표시 = 자동 추천)
4. 트랙에 삽입을 클릭하면 악기별로 새 트랙이 추가됩니다.

이 기능을 사용하려면 Python 패키지(oemer, music21 등)와 FluidSynth, SoundFont(.sf2)가 필요합니다. 아래 필수 도구 설치 — ⑦⑧⑨ 를 참고하세요.

내보내기

파일 → WAV로 내보내기 (Ctrl+E)로 전체 타임라인을 24-bit WAV로 렌더링합니다. 뮤트된 트랙은 출력에 포함되지 않습니다. 메트로놈이 켜져 있으면 출력에 포함되므로 내보내기 전에 확인하세요.

필수 도구 설치

스템 분리(MR 추출)와 노이즈 감소 기능을 사용하려면 아래 도구를 설치해야 합니다. 기본 WAV 편집과 내보내기는 추가 설치 없이 사용 가능합니다.

① Python 설치 — 필수 (MR 추출 전체)

python.org/downloads에서 Python 3.10 이상을 다운로드하세요. 설치 시 반드시 "Add Python to PATH" 옵션을 체크하세요. 설치 완료 후 셀라를 재시작하세요.

⚠ Windows 스토어(Microsoft Store) Python은 사용하지 마세요. 백그라운드 실행 시 제대로 동작하지 않습니다.
설치 확인 (명령 프롬프트):
python --version

② ONNX Runtime 설치 — 필수 (Demucs 모델)

pip install onnxruntime numpy scipy
pip 명령을 찾지 못하는 경우:
python -m pip install onnxruntime numpy scipy
⚠ 가상 환경(venv, conda) 사용 시: 셀라가 가상 환경을 자동 인식하지 않습니다. 전역 Python에 설치하거나 py -m pip install을 사용하세요.

③ audio-separator 설치 — 권장 (보컬 분리)

Demucs보다 훨씬 넓은 범위의 목소리를 인식합니다. GPU 가속을 원하면 [cpu] 대신 [gpu]를 사용하세요. (CUDA 12.x 필요)

pip install "audio-separator[cpu]"

④ noisereduce 설치 — 필수 (노이즈 감소)

pip install noisereduce

⑤ FFmpeg 설치 — 필수 (MP3·MP4 임포트)

방법 A — 명령 프롬프트를 관리자 권한으로 열고 (권장):

winget install Gyan.FFmpeg

방법 B (수동) — ffmpeg.org/download.html에서 Windows 빌드를 다운로드하고, bin\ 폴더 경로를 시스템 PATH 환경 변수에 추가합니다.

⑥ AI 분리 모델 다운로드

위 도구 설치 후, 도구 → 모델 관리자에서 AI 모델을 다운로드합니다.

Hybrid Transformer Demucs (htdemucs)
4스템 분리 · 303 MB · ONNX Runtime 필요
UVR MDX-NET Vocal FT
보컬/반주 2스템 · 65 MB · audio-separator 필요

모델 선택 후 모델 설치 / 다운로드를 클릭하고, 완료 후 새로고침을 누르세요.

⑦ 악보 인식(OMR) 패키지 — 필수 (악보 가져오기)

pip install oemer music21 mido Pillow scipy
oemer는 첫 실행 시 자체 모델을 자동 다운로드합니다 (인터넷 연결 필요).

⑧ FluidSynth 설치 — 필수 (악보 → 오디오 합성)

두 단계가 모두 필요합니다.

Step 1 — 네이티브 라이브러리 설치 (libfluidsynth-3.dll):

choco install fluidsynth

또는 fluidsynth.org에서 Windows 설치 프로그램을 다운로드하여 실행합니다. 셀라가 DLL을 자동 탐지하므로 PATH 설정은 불필요합니다.

Chocolatey는 버전에 따라 C:\Program Files\FluidSynth\bin\ 또는 C:\tools\fluidsynth\bin\에 설치할 수 있습니다. 셀라는 두 경로를 모두 자동으로 탐지합니다.

Step 2 — Python 래퍼 설치:

pip install fluidsynth
⚠ pip만 설치하고 네이티브 설치 프로그램을 생략하면 합성 시 오류가 발생합니다. 반드시 두 단계를 모두 수행하세요.

문제 해결 — "FLUIDSYNTH_DLL_MISSING" 오류: 합성 시 이 오류가 발생하면 셀라가 libfluidsynth-3.dll을 찾지 못한 것입니다. 관리자 PowerShell에서 아래 명령으로 DLL 위치를 확인하세요:

Get-ChildItem -Path C:\ -Filter "libfluidsynth*.dll" -Recurse -ErrorAction SilentlyContinue | Select-Object -ExpandProperty FullName

DLL이 발견되면 해당 폴더(예: C:\tools\fluidsynth\bin)를 시스템 PATH에 추가한 뒤 셀라를 재시작하세요.

⑨ SoundFont(.sf2) 설치 — 필수 (악기 음색)

FluidSynth가 MIDI를 실제 악기 음색으로 변환하는 악기 샘플 뱅크입니다.

추천 무료 SoundFont: GeneralUser GS (~29 MB)
schristiancollins.com/generaluser.php 에서 다운로드 후
GeneralUser GS.sf2 파일을 아래 경로에 복사합니다:

%AppData%\Selah\soundfonts\
탐색기 주소 표시줄에 위 경로를 그대로 붙여넣으면 폴더가 열립니다.

윤리적 사용 가이드라인

Selah는 예배 사역 지원을 위해 제작되었습니다. 처리 대상 음원의 저작권 상태는 사용자가 직접 확인할 책임이 있습니다.

프로젝트 목적

Selah는 기독교 예배 준비, 소형 교회 및 개척교회 오디오 작업, 선교 및 사역 관련 반주 준비를 지원하기 위해 개발된 사역 지향 오디오 소프트웨어입니다. 비상업적·섬김 중심·로컬 환경 중심의 활용을 우선합니다.

권리와 소유에 대한 존중

작곡자, 작사가, 연주자, 녹음물을 제작한 교회와 사역 단체, 출판사 및 저작권자의 권리를 존중해야 합니다. 기술적으로 가능하다고 해서 법적·윤리적으로 허용되는 것은 아닙니다.

책임 있는 사용 예시

✓ 적법하게 접근 가능한 예배 음원을 소규모 찬양팀 반주 준비에 활용
✓ 교회 내부 리허설을 위해 사역 녹음물을 정리
✓ 자신이 처리 권한을 가진 오디오 자료에 대해 스템 분리 시험

문제가 될 수 있는 사용 예시

✗ 허락 없이 저작권 있는 예배 녹음물을 재배포
✗ 분리된 스템을 상업적으로 재포장
✗ 적법한 권리 없이 보컬 제거·트랙 추출
✗ 특정 교회·사역 단체의 승인 없이 승인된 것처럼 오인시키는 사용

AI / 모델 관련 주의

모델 출력은 완벽하지 않을 수 있으며 아티팩트·재구성 오류가 포함될 수 있습니다. 제3자 모델 약관이 추가 제한을 둘 수 있습니다. 유지관리자는 상업 배포·공개 릴리스에 대한 적합성을 보장하지 않습니다.
