## v1.0.0 (2026-04-17)

첫 번째 안정화 릴리즈.

### 신규 — 클립 편집

- **다중 클립 선택** — Ctrl+클릭으로 개별 클립 선택 토글, Shift+클릭으로 범위 선택
- **클립보드 작업** — 복사(Ctrl+C), 잘라내기(Ctrl+X), 붙여넣기(Ctrl+V)
- **클립 합치기** (Ctrl+M) — 같은 트랙에서 선택된 클립들을 하나로 합침
- **선택 클립 분할** (S) — 선택된 모든 클립을 플레이헤드 위치에서 동시에 분할
- **앞 클립 바로 뒤로 이동** (Ctrl+J) — 선택 그룹을 같은 트랙의 바로 앞 클립 끝에 붙여 이동
- **플레이헤드 위치로 이동** (Ctrl+G) — 선택 그룹의 첫 클립이 플레이헤드 위치에서 시작하도록 이동
- **트랙 시작 위치로 이동** (Ctrl+H) — 선택 그룹의 첫 클립이 위치 0에서 시작하도록 이동

### 신규 — 트랜스포트

- 자동 스크롤: 재생 중 플레이헤드가 화면을 벗어나면 타임라인이 자동으로 따라감

### 버그 수정

- 정지+처음으로 이후 재생 시간 카운터가 00:00.000에서 멈추는 문제 (원인: `Seek()` 호출 시 `_lastPlayheadFrames` 미초기화)
- 타임라인 눈금자 클릭으로 탐색해도 `PlayheadTimeDisplay`가 갱신되지 않는 문제
- 정지+처음으로 후 재생/정지 버튼 상태가 잘못 표시되는 문제 (`Dispatcher.BeginInvoke`와 오디오 재시작 간 경쟁 조건)
- 이전 재생 세션의 오래된 `PlayheadAdvanced` 이벤트가 초기화된 플레이헤드 위치를 덮어쓰는 문제 (재생 세대 카운터로 해결)
- Ctrl+J(앞 클립 뒤로 이동) 실행 후 마우스 클릭 전까지 화면에 반영되지 않는 문제 (캔버스가 `ClipViewModel.PropertyChanged`를 구독하지 않던 문제)

## v0.9.0 (2026-04-16 ~ 2026-04-17)

### 신규 — 악보 인식

- 스캔·촬영한 악보 이미지를 악기별 오디오 트랙으로 가져오기
- OMR 파이프라인: oemer(광학 악보 인식) → MusicXML → MIDI → FluidSynth 합성 → WAV
- SF3 SoundFont 형식 지원 (SF2에 추가)
- 악보 내 템포맵을 반영한 가변 템포 렌더링
- 어두운 배경(반전) 악보 이미지 자동 보정
- 투명(RGBA/LA/P) 이미지를 흰 배경에 합성 후 처리
- 다양한 FluidSynth 설치 경로 자동 탐색 (Chocolatey, 수동 설치, PATH 등)
- 인식 실패 시 원인과 해결 방법을 안내하는 친화적 오류 메시지
- 악보 인식 다이얼로그 내 지원 악기 안내 패널
- 대용량 악보 처리 시 메모리 급증 및 UI 지연 개선

## v0.8.0 (2026-04-14 ~ 2026-04-16)

### 신규 — 스템 분리 및 노이즈 제거

- **스템 분리** — 세 가지 엔진 지원:
  - audio-separator (MDX-Net, 기본 — 보컬 분리 품질 최우수)
  - ONNX Runtime (경량, GPU 불필요)
  - Demucs (GPU 가속, 최고 품질)
- **노이즈 제거** — noisereduce 스펙트럼 게이팅 (무료, 로컬, GPU 불필요)
- 모델 관리자 창 — ML 모델 다운로드 및 관리, 실시간 진행 표시
- 스템 분리 중 실시간 트랙 생성
- 가져오기·내보내기·스템 분리·노이즈 제거 작업 진행 팝업 창
- Python 또는 FFmpeg 미설치 시 시작 시 설치 안내 표시

## v0.1.0 (2026-04-08 ~ 2026-04-11)

### 최초 구현

- 핵심 데이터 모델: Project → Track → Clip → AudioSource, gzip 압축 JSON(`.slh`)으로 저장
- WPF 타임라인 캔버스 — 웨이브폼 렌더링, 스크롤·줌 지원
- 재생 엔진: WaveOut(기본) + WASAPI Shared(폴백), 마스터 믹서 + 트랙별 게인/패닝/솔로/뮤트
- 소프트 리미터, 템포맵 기반 메트로놈
- FFmpeg를 통한 오디오 가져오기·내보내기, 하드웨어 감지 서비스
- 클립 재생 완료 시 자동 정지
- 비파괴 편집 — 클립은 소스 파일의 구간을 참조하며 원본 파일은 절대 수정하지 않음

### 오디오 안정성 수정 (이 단계에서 적용)

- 반복 재생/정지 시 교착 상태(데드락) 발생
- WASAPI 샘플레이트 불일치로 인한 저음 잡음
- SoftLimiter 니 영역 계산 오류로 인한 왜곡 노이즈
- 읽기 중 탐색(seek-on-read)으로 인한 오디오 노이즈
- 플레이헤드 시간 표시에 밀리초 대신 리터럴 `f` 문자가 출력되는 문제
